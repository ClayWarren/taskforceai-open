# Security Policy

## Reporting a Vulnerability

Please do not report suspected vulnerabilities in a public issue.

Email security@taskforceai.chat with a description, reproduction steps, impact,
and any suggested remediation. We aim to acknowledge reports within 24 hours
and provide an initial assessment within 72 hours.

Useful reports include authentication or authorization bypasses, cross-tenant
access, remote code execution, server-side request forgery, secret disclosure,
webhook or update-signature bypasses, and sandbox or workspace-boundary escapes.

User-authorized agent actions within a selected workspace or through explicitly
configured integrations are expected behavior unless they cross a documented
permission, account, workspace, or network boundary.

## Supported Versions

Security fixes are applied to the latest public release and the default branch.
Please verify reports against current code when practical.
